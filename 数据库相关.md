# 二、PostgreSQL的相关知识点

## PGSQL的Database、Schema和Table

### **Database（数据库）**

#### **MySQL:**

在 MySQL 中：

- **Database（数据库）** 是最顶层的对象，包含多个表（`tables`）以及其它数据库对象。
- 每个数据库有自己的数据存储、索引、日志文件等。
- 使用 `CREATE DATABASE` 可以创建一个新的数据库。

```sql
CREATE DATABASE my_database;
```

- 数据库之间互相隔离，不共享数据。

#### **PostgreSQL:**

在 PostgreSQL 中：

- **Database（数据库）** 的概念与 MySQL 相似，是最顶层的对象，包含所有数据库对象（如：`Schemas`、`Tables` 等）。
- PostgreSQL 不允许跨数据库查询。每个数据库都是完全隔离的。
- 使用 `CREATE DATABASE` 创建数据库。

```sql
CREATE DATABASE my_database;
```

- 每个 PostgreSQL 数据库可以有多个 **Schema**，而 MySQL 数据库则没有类似的概念，只有直接的表（Table）。

#### **总结：**

在两者中，`Database` 的概念是相同的，都是一个逻辑容器，包含所有的数据库对象。

------

### **Schema（模式）**

#### **MySQL:**

- 在 MySQL 中，没有单独的 **Schema** 这个概念。它使用 **Database** 来代替 **Schema**。
- 你可以将数据库视为 schema，但实际上数据库和 schema 是完全相同的。每个 MySQL 数据库就是一个 schema，没有更细粒度的划分。

```sql
-- 这在 MySQL 中是合法的，因为 Database 作为 Schema 使用
CREATE DATABASE my_schema;
```

- 在 MySQL 中，当你创建一个数据库时，它直接包含所有的表、视图、存储过程等。

#### **PostgreSQL:**

- 在 PostgreSQL 中，**Schema** 是一个逻辑结构，用来组织数据库中的表、视图、函数等对象。每个数据库中可以有多个 schema。
- 默认情况下，PostgreSQL 中的数据库会有一个名为 `public` 的默认 schema，所有未指定 schema 的表都会创建在 `public` schema 中。
- 你可以使用 `CREATE SCHEMA` 来创建一个新的 schema。

```sql
CREATE SCHEMA my_schema;
```

- 你可以使用 `SET search_path` 来指定当前会话中默认的 schema。例如，在一个数据库中有多个 schema，你可以设置默认的 schema 来避免每次都写 `schema_name.table_name`。

```sql
SET search_path TO my_schema;
```

- **Schema** 在 PostgreSQL 中的主要用途是逻辑分隔，使得多个应用或用户可以在同一个数据库中创建并管理他们自己的表和对象，而不会产生冲突。

#### **总结：**

- **MySQL** 没有明确的 `Schema` 概念，使用数据库作为最顶层对象。
- **PostgreSQL** 中，`Schema` 是一个组织数据库对象的逻辑结构，一个数据库可以包含多个 schema。

------

### **Table（表）**

#### **MySQL:**

- 在 MySQL 中，**表（Table）** 是存储数据的地方，所有的数据都以行和列的形式保存在表中。
- 表在 MySQL 中创建时通常指定在某个数据库下，使用 `CREATE TABLE` 来定义。
- 在 MySQL 中，表属于某个特定的数据库，通常以 `database_name.table_name` 的方式来访问。

```sql
CREATE TABLE my_table (
  id INT PRIMARY KEY,
  name VARCHAR(255)
);
```

#### **PostgreSQL:**

- 在 PostgreSQL 中，**表（Table）** 的概念与 MySQL 相同，它是存储数据的地方，按行列的形式存储数据。
- 表可以存在于不同的 schema 下，每个 schema 可以有相同名称的表，因此你需要指定 schema 来区分它们。
- PostgreSQL 的表创建语法与 MySQL 类似，使用 `CREATE TABLE`。

```sql
CREATE TABLE my_schema.my_table (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255)
);
```

- 访问时，如果没有指定 schema，默认使用 `public` schema。例如，你可以通过 `my_table` 来访问表（如果它在 `public` schema 下），但如果表在其它 schema 下，你必须使用 `schema_name.table_name` 这种方式来引用它。

#### **总结：**

- 在 **MySQL** 中，表直接属于数据库，没有像 PostgreSQL 那样的 schema 概念。
- 在 **PostgreSQL** 中，表属于 schema，一个数据库可以有多个 schema，每个 schema 下可以有多个表。

------

### 总结对比：

| 概念         | MySQL                                  | PostgreSQL                                                   |
| ------------ | -------------------------------------- | ------------------------------------------------------------ |
| **Database** | 数据库是顶层容器，包含所有数据表。     | 数据库是顶层容器，包含多个 schema。                          |
| **Schema**   | 没有独立的 schema，数据库即为 schema。 | schema 是组织数据库对象的逻辑容器，一个数据库可以有多个 schema。 |
| **Table**    | 表直接存在于数据库中。                 | 表属于 schema，一个数据库可以有多个 schema，每个 schema 可以有多个表。 |

### PostgreSQL 特有的用法：

1. **Schema 的作用：**
   - Schema 在 PostgreSQL 中用于逻辑分隔对象，例如在同一个数据库中不同的用户或应用程序可以使用不同的 schema 来存放他们的表、视图、函数等，从而避免冲突。
2. **表的命名：**
   - PostgreSQL 中可以通过 schema 来组织表。访问表时可以使用 `schema_name.table_name`，如果不指定 schema，默认使用 `public` schema。
3. **多 schema 的好处：**
   - 多 schema 可以帮助管理复杂的数据库结构，在同一个数据库中存储不同应用或团队的数据，避免对象命名冲突，提升管理效率。
